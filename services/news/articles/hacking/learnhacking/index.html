Want to become a hacker? Then this guide will help you to get started!

What is Hacking?

Hacking refers to activities that seek to compromise digital devices, such as computers, smartphones, tablets, and even entire networks. Hacking is a Art. As a hacker you think outside the box. It is not always about the tools. Like Darren Kitchen of Hak5 once said: "Hacking is a Way of Thinking."

To get a basic understanding of hackers, you need to know about the different types of hackers.

Types of Hackers

Here are the most populair types:

White Hat Hackers
The Good Guys. Also known as Ethical Hackers. White Hats Hackers often use the same techniques and methods of hacking as Black/Grey Hats. But they have permission to do it.

Black Hat Hackers
The Bad Guys. Blacks Hats hack in order to gain unauthorised access to systems without permission of the owner. Black Hat hacking is always illegal because of its bad intent which includes stealing corporate data, violating privacy, damaging the system, blocking network communication, etc.

Grey Hat Hackers
Grey hats are a blend of both black hat and white hat hackers. Often, grey hat hackers will look for vulnerabilities in a system without the owner’s permission or knowledge. Vulnerabilities that are found will be reported to the owner.

Red Hat Hackers
Often known as the Do Whatever They Want" hackers. They are the vigilantes of the hacking world. They basically live and play by their own rules. They have the same intentions as White Hat Hackers, but they do it their own way.

Script Kiddies
Often known as: skiddie. There is no hat given for Script Kiddies, because they have no real alliance to the hacker world. They don't care about the Art of Hacking. They usually download an overly used tool and learn how to use it. Often involved in DDoS attacks.

Setup Your Own Hacking Lab

In this tutorial we will be using Kali Linux - A Advanced Penetration Testing Distribution.

Kali Linux
Install: https://www.kali.org/downloads/
Choose the Kali Linux 64-Bit (Installer).

Virtual Platforms

There are many ways to setup a Hacking Lab, but the safest way for beginners is by using a Virtual Platform like VMWare and VirtualBox.

VirtualBox
Install: https://www.virtualbox.org/
1. Launch Oracle VM VirtualBox and click New in the VirtualBox Manager window to create a Virtual machine.
2. A wizard is shown, to guide you through setting up a new virtual machine (VM).
3. Now you need to import the Kali ISO that you just downloaded.
4. Choose how much memory to allocate to the virtual machine and click Next. Default size: 1024 MB.
5. The default option is to create a Virtual Hard Disk [VHD] for the new VM. Click Create to continue.
6. Stick to the default file type for the new virtual hard disk, VDI (VirtualBox Disk Image).
7. Storage on a physical hard disk. Decide between Dynamically allocated and Fixed size.
*The first choice allows the new hard disk to grow and fill up space dedicated to it.
*The second, fixed size, uses the maximum capacity from the start.
8. File location and size. Specify the name and where you want to store the virtual hard disk. Choose the amount of file data the VM is allowed to store on the hard disk. We advise giving it at least 8 GB. Click Create to finish.

ZeroMQ protocol

Each server runs an agent (minion), which connects to a master. A master is a Salt installation that collects state reports from minions and publishes update messages that minions can act on. In order to communicate, the master uses two ZeroMQ channels. One is the “request server” where minions connect to report their status. And the other one is the “publish server” where the master publishes messages that the minions can connect and subscribe to.

The authentication bypass can be achieved because the ClearFuncs class processes unauthenticated requests and unintentionally exposes the _send_pub(). This is the method used to queue messages from the master publish server to the minions.

As for the directory traversal, the so called “wheel” module contains commands used to read and write files under specific directory paths.

The inputs to these functions are linked with the target directory and the resulting path is not canonicalized. The get_token() method of the salt.tokens.localfs class fails to sanitize the token input parameter which is used as a filename, allowing the reading of files outside of the intended directory.

What is RCE?

Remote code execution (RCE) is the ability an attacker has to access someone else's computing device and make changes, no matter where the device is geographically located.

Vulnerabilities can provide an attacker with the ability to execute malicious code and take complete control of an affected system with the privileges of the user running the application. After gaining access to the system, attackers will often attempt to elevate their privileges.
